SMED 深化
讓快速換線變成可複製的能力

單分鐘換模（Single-Minute Exchange of Die）的下一步：穩定、可複製、可擴散

01 ｜核心定義

一句話講清楚 SMED

把換線工作拆成「內部 / 外部」，盡量把內部作業（停機才能做）外部化，再把剩下的內部作業 簡化、並行化、標準化，以縮短停機時間。

🔴 內部作業

必須停機才能做的事
（拆模、上模、定位）

🟢 外部作業

機器仍在運轉時就能準備的事
（備料、備工具、預組）

02 ｜深化六層次

從淺到深，SMED 深化的 6 個層次

① 一次改善 → 標準

把成果固化為 SOP

② 主觀 → 數據化

量測明細時間

③ 單點 → 系統性

處理源頭浪費

④ 技術 → 同步

供應 / 物流 / 資訊

⑤ 快 → 一次到位

首件 OK 率

⑥ 單機 → 複製

擴散到產品族群

層次 ①

從一次改善 → 變成標準

  • 改善不是「做完就結案」，而是要固化。
  • 把新方法寫成 SOP點檢表OJT 教學法。
  • 確保不同班別、不同人都照同一套做，減少人因變異。

📌 深化重點：沒有標準，就沒有改善的延續。

層次 ②

從主觀估計 → 數據化管理

別只看「總換線時間」，要拆解成可分析的明細：

📊 必要量測項

  • 內部 / 外部時間
  • 等待時間
  • 找工具時間
  • 調整時間
  • 首件確認時間

🎯 為什麼這樣拆

看得到才改得動。把「黑盒子」拆成可比較的小區塊，才有改善方向。

層次 ③

從單點改善 → 系統性消除浪費

不只改善動作，要處理造成浪費的源頭：

🔧 治具設計

夾治具是否好拆好裝？

📍 夾治具定位

是否需要量測對位？

📦 材料供應方式

是否能線邊就位？

✅ 品質判定方式

首件能否一次過？

層次 ④

從技術改善 → 供應／物流／資訊同步

外部化要成功，常常要靠「跨單位提前到位」：

  • 物料預先備料、線邊定位
  • 前段製程提前準備、提前送達
  • 系統提前開工單、文件提前就緒
  • 工具由備料人員預先點檢、上車

👉 SMED 深化的成功＝跨部門協作的成功。

層次 ⑤

從縮短換線 → 提升「一次到位」

換線快但首件不穩 / 反覆調機 / 報廢增加＝沒有真正改善。

必納入 KPI

  • 首件一次 OK 率
  • 調整次數
  • 換線後報廢率

背後的觀念

速度只是手段，穩定的速度才是真本事。

層次 ⑥

從單機台 → 複製到產品族群／產線

把方法抽象成可複製的「換線模組」：

🛠 通用工具車

每族群一格、影子板定位

🔌 通用治具接口

統一介面、隨插即用

📐 通用定位基準

一次定位、不靠師傅手感

📑 通用檢查規格

首件判定可量化

03 ｜落地範例

三個現場能直接做的範例

A. 換線工具車

原狀況：換線才找工具
做法：每族群一格＋影子板，備料人員前 10 分鐘點檢→外部化
效果：找工具時間趨近 0

B. 機構防呆＋基準化

原狀況：靠師傅微調對位
做法：定位銷、止擋、Poka-Yoke、一次定位設計
效果：調整↓ 首件 OK↑

C. 並行作業設計

原狀況：一人按序做完才換下一步
做法：A 拆舊模、B 同時預組新模
效果：總停機時間明顯下降

04 ｜帶走重點

衡量指標 ＆ 行動清單

📈 衡量指標

  • 停機換線時間（內部時間）
  • 外部作業完成率
  • 首件一次 OK 率 / 調整次數
  • 不同班別、不同人的時間變異

🚀 下一步行動

  • 選一條產線做基線量測
  • 列出可外部化的項目清單
  • 製作工具車與點檢表
  • 加入定位防呆設計
  • 把成果固化為 SOP，並擴散

速度只是手段，穩定的速度才是真本事。
